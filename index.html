Bots For Hire

AI Employee Hiring App

  • React
  • SCSS
  • API

You can forget human error, take a step into the future of business with your next hire.

Project Overview

Project Description

Employers can browse the endless amount of artificial intelligence for hire making the whole hiring process quick and easy. The program utilizes API fetches to grab employee information containing: Name, job title, key skill, email, phone number, and location. Each time User clicks the "Find Bots" button, there will be a fetch request to grab four new users and render them on screen for the user. There is also an option to limit the number of rendered users by utilizing the text box and inputting a number between 1 and 4.

Why did I build this project?

I built this project to get a stronger grasp on React. I have designed this project in a way that will allow me to continously add new features. These new features will be those that can be used for various kinds of projects, I'm specfically trying to add new features that would commonly be seen on e-commerce sites. This project is the perfect amount of beginner friendly while leaving room for advanced features as my skills improve. I have been having a blast creating this project and look forward to continously add more to it.

Build Difficulties?

Understanding how to properly pass state through components has had its ups and downs but also led me to beneficial resources with methods to make my roadblocks quick and painless. In the beginning, it was fairly straightforward to follow the logic, as my app grew, it was made clear that I needed to find strategies to properly maintain state for occasions when multiple components relied on the same state. This has led me to learning useContext has been great at passing state between components. Nesting components with one another had given me a few roadblocks but it was fun to solve where I was going wrong. Each roadblock pointed me towards great resources that I will use for my future projects.

Pyzza Bytes

FOOD ORDERING APP

  • HTML5
  • CSS3
  • JavaScript
  • Bootstrap

Pyzza Bytes is a food ordering app that lets customers order delicious food with ease.

Project Overview

Project Description

User can browse the wide range of items and add them to their cart. User must select an option for all form sections to add the item to their order. Program will alert the user to fill out all information if they don't. Each item added to order will update the current total displayed and update the "View Order" modal window. The modal window displays the item description, sub total, tax, and total cost. User has the option to close the modal window or submit their order. A order submission will be greeted with a second modal window notifying the customer that the order has been submitted successfully. User has the option to start a new order after submission which will do a global reset

Why did I build this project?

I built this project for learning experience. I wanted to make a challenging project that would allow me to continously add more features to it as my skills improve. This project allows me to get as close to a real world project as possible. I learned so much after implementing a few of the features into this project and I'm excited to see where it takes me a few months from now.

Build Difficulties?

Implementing a nested modal window to display the current order. This became our first real challenge as I was still getting familiar with utilizing bootstrap components. It took a bit of time to think about how I wanted to add the order list and a second modal window to display the order submission. Luckily, the bootstrap documentation is very thorough and easy to read, I was able to come up with a clean layout.

Future Features:

  • Remove item from order
  • Discount for certain items based on current day of the week
  • Create an account
