Pyzza Bytes

FOOD ORDERING APP

  • HTML5
  • CSS3
  • Vanilla JS
  • Bootstrap

Pyzza Bytes is a food ordering app that lets customers order delicious food with ease.

Project Overview

Project Description

User can browse the wide range of items and add them to their cart. User must select an option for all form sections to add the item to their order. Program will alert the user to fill out all information if they don't. Each item added to order will update the current total displayed and update the "View Order" modal window. The modal window displays the item description, sub total, tax, and total cost. User has the option to close the modal window or submit their order. A order submission will be greeted with a second modal window notifying the customer that the order has been submitted successfully. User has the option to start a new order after submission which will do a global reset

Why did I build this project?

I built this project for learning experience. I wanted to make a challenging project that would allow me to continously add more features to it as my skills improve. This project allows me to get as close to a real world project as possible. I learned so much after implementing a few of the features into this project and I'm excited to see where it takes me a few months from now.

Build Difficulties?

Implementing a nested modal window to display the current order. This became our first real challenge as I was still getting familiar with utilizing bootstrap components. It took a bit of time to think about how I wanted to add the order list and a second modal window to display the order submission. Luckily, the bootstrap documentation is very thorough and easy to read, I was able to come up with a clean layout.

Future Features:

  • Remove item from order
  • Discount for certain items based on current day of the week
  • Create an account

Redline Detailing

APPOINTMENT SCHEDULING APP

  • HTML5
  • CSS3
  • Vanilla JS

Redline Detailing is the ultimate detailing service that leaves no stone unturned when it comes to taking care of your car.

Project Overview

Project Description

This is a static site created primarily with HTML and CSS with light touches of JavaScript. The site's design shows potentional clients what the site can look like when complete. The website was designed to have a sleek design that is eye catching yet minimalistic. I chose to go in this direction to keep the attention on the package content and less distractions to make the process less intimidating for customers. The package sections have a glow catch the customers attention and content is laid out in an easy-to-read manner. A testimonials page is included to ensure the company is reputable.

Why did I build this project?

This was a learning project to get additional practice with CSS. I wanted to simulate a mockup for a potential client by adding sections such as: testimonial section, schedule form, email quote section. While not fully functional, the thought process was to simulate how the site can look like without full commitment to the design incase the client wanted to go a different direction. After creating my previous site(Echo Trails), I was able to focus more on the advanced CSS layouts by utilizing grid and flexbox for my container content. These two tools together made creating layouts effortless. I was also able to create a unique design utilizing shadows that really made the package section stand out.

Build Difficulties?

Having my hero image and other site images look good became a bit of a challenge in the beginning but I realized my mistake and by adjusting the dimensions into viewport width and height, solved my problem.

Future Features:

  • Cancel an existing appointment
  • Update an existing appointment
  • View current appointment
  • Create an account

Echo Trails

OUTDOOR ADVENTURE APP

  • HTML5
  • CSS3

Echo Trails makes planning an exciting adventure effortless by bundling your favorite outdoor activities all into one package.

Project Overview

Project Description

This was one of our first real projects after completing our online course to test what we have learned. This site was made primarily with HTML and CSS. Being our first large site, we spent a good portion of time using documentation and other online resources to progress through the site. By putting away the tutorials, we were able to quickly see which areas we needed to work on. During the journey I grew to love using MDN Web Docs and chrome DevTools. These tools had made creating layouts so much easier and helped me better understand good and bad practices. Even through the struggles, I loved building a simple idea in my head, into a website that didn't exist before. After completing the project, I learned so much more through building the project than I had through tutorials.

Why did I build this project?

I built this project to put what I learned into practice by putting away tutorials and testing my knowledge on the topics covered. Being my first page, I spent a good amount of time reading documentation as I was going through the process of creating each section.

Build Difficulties?

Learning flexbox and grid were a bit challenging to grasp in the beginning. With the use of MDN docs, they started to make sense and I came to love utilizing these great tools for my site layouts.

Media Queries were challenging to do by hand. I had to read documentation to better understand them and once I implemented my media query designed for desktop, it all started to come together. I enjoyed learning how media queries operate behind the scenes.
