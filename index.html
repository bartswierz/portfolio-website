Civilience

Community Health Tracker App

  • React
  • CSS
  • AWS Amplify
  • AWS DynamoDB
  • Amplify Authentication
  • Context Hooks
  • Redux Hooks

As a Front-end co-team lead at Civilience, I collaborated closely with the UI/UX Design team to build a complex health tracker web application from scratch by translating Figma designs into working code. The app is intended to be used by communities across the US to help them understand their emotional health.

Project Overview

Helping individuals and communities gain a better understanding of their emotional health by taking them through a flow of questions and process screens to see how all their feelings and actions are aligned. Not only will user's be able to better understand their own emotions but also be able to see how communities all over the US are based on zipcode.

This Emotional Health Tracker(Feelings Flow Tracker) was created to help communities and individuals understand their emotional health. By taking the survey daily, the individual will gain new vocabulary to better describe their emotional health and be taken through the flow of the steps to see how their actions and feelings align. This app is intended to be used across the US by thousands or quite possibly millions as Civilience gains traction as Civilience is a relatively new great non-profit organization.

Being new to AWS, it required reading through a lot of the AWS documentation and researching how to implement all the great technologies it has to offer. With the help from our supervisor, the frontend team was able to figure out how to utlize Amplify Authenticator and data storage to DynamoDB. The Amazon documentation was extremely helpful in figuring out all the difficult setup to utilize the technologies. It was a slow process but the frontend team was able to create a good foundation for the application based on the Figma Designs and slowly work towards making those designs into reality. Another difficulty experienced was taking on the role of a Front-end co-team lead. My reponsibilities included task delegation and task management ensuring all frontend tasks were met by the deadlines and of high quality. Using my previous management skills from past employment, I was able to quickly organize a task list and divide it up according to where the team members were working on within our codebase. We wanted to avoid stepping over each others toe and tackle different areas of the codebase simultaneously. This strategy was successful and helped us stay on track to meet each deadline.

Team collaboration is such an important skill to have as a developer, I am very thankful for experiencing this at Civilience. This application while it seemed simple at first, required constant collaboration across multiple teams(UI/UX Design, Front-end & Data Science) as the app changed shaped constantly so it was a must to be on top of it all. I was able to experience the importance of cross-team meetings to ensure everyone was on the same page, be aware of difficulties surfaced, and so much more. My role as a Frontend Co-team lead required me to be aware of all recent UI/UX Design changes by collaborating with the UI/UX Team leads so I could effectively distribute tasks to frontend team members for maximum efficiency. My prior management skills were put to great use and was able to further improve upon them as we progressed closer to completion each week. Another area I improved upon was collaborating with supervisors, managers, and other upper management roles during team meetings. I took it upon myself to do the weekly front-end demos for the other teams and answered any questions that came up. I enjoyed being able to help the UI/UX Design Team by being their main contact for any questions they may have regarding what was possible to build and advice to help make our application mobile friendly.

Pyzza Bytes

FOOD ORDERING APP

  • HTML5
  • CSS3
  • JavaScript
  • Bootstrap

Pyzza Bytes is a food ordering app that lets customers order delicious food with ease.

Project Overview

User can browse the wide range of items and add them to their cart. User must select an option for all form sections to add the item to their order. Program will alert the user to fill out all information if they don't. Each item added to order will update the current total displayed and update the "View Order" modal window. The modal window displays the item description, sub total, tax, and total cost. User has the option to close the modal window or submit their order. A order submission will be greeted with a second modal window notifying the customer that the order has been submitted successfully. User has the option to start a new order after submission which will do a global reset

I built this project for learning experience. I wanted to make a challenging project that would allow me to continously add more features to it as my skills improve. This project allows me to get as close to a real world project as possible. I learned so much after implementing a few of the features into this project and I'm excited to see where it takes me a few months from now.

Implementing a nested modal window to display the current order. This became our first real challenge as I was still getting familiar with utilizing bootstrap components. It took a bit of time to think about how I wanted to add the order list and a second modal window to display the order submission. Luckily, the bootstrap documentation is very thorough and easy to read, I was able to come up with a clean layout.
